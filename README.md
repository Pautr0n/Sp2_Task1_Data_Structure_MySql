# Sprint 2, Task 1, Data Structures and MySql

## Exercise Description:

### ***Level1:***
Database modeling for two business scenarios:
- Optical Store "Cul d'Ampolla": manages customers, employees, suppliers, and eyeglass sales.
- Online Pizzeria: handles customer orders, product categories, store locations, staff roles, and delivery tracking.


### ***Level2:***
Design of a simplified database for a reduced version of YouTube:
- Users, videos, channels, tags, likes/dislikes, playlists, comments, and user interactions.


### ***Level3:***
Modeling the database for a simplified version of Spotify:
- Free and premium users, subscriptions, payments, playlists, songs, albums, artists, favorites, and artist relationships.


## **Technologies Used**
- MySQL Workbench 8.0 CE: Used for designing and modeling the relational database schema (ER diagrams).
- MySQL Server 8.0+: Required to run and test the database locally.
- SQL (Structured Query Language): Used to define tables, relationships, constraints, and sample queries.
- Git & GitHub

## **Requirements**
- MySQL Workbench 8.0 CE installed
- MySQL Server 8.0 or higher running locally or remotely
